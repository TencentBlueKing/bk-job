# 回答用户时带上日志平台查询链接

## 概述

当完成蓝鲸作业平台任务日志查询并回答用户时，必须在回答中包含以下关键信息：
1. 任务的 `request_id`
2. 日志平台的查询链接

## 回答格式规范

每次回答用户关于作业平台任务的查询时，回答内容应包含以下结构：

```
1. 任务信息摘要
   - request_id: xxx
   
2. 分析报告/用户需求的解答
   - （根据用户具体需求提供分析内容）

3. 日志平台查询链接
   - 提供可直接访问的日志平台链接，方便用户自行查看完整日志
```

## 日志平台链接生成规则

### 链接模板

```
https://bklog.example.com/#/retrieve/481?projectId=6&bizId=7&start_time=now-7d&end_time=now&interval=auto&search_mode=sql&keyword=request_id%253A%2520%2522{REQUEST_ID}%2522
```

### 参数说明

| 参数          | 值                                          | 说明                |
|-------------|--------------------------------------------|-------------------|
| projectId   | 6                                          | 日志平台项目ID（固定值）     |
| bizId       | 7                                          | 业务ID（固定值）         |
| start_time  | now-7d                                     | 查询起始时间，7天前        |
| end_time    | now                                        | 查询结束时间，默认当前时间     |
| interval    | auto                                       | 时间间隔，自动           |
| search_mode | sql                                        | 搜索模式              |
| keyword     | request_id%253A%2520%2522{REQUEST_ID}%2522 | 查询关键词，需要进行 URL 编码 |

### 链接生成步骤

1. 获取任务的 `request_id`
2. 将 `request_id` 替换到链接模板中的 `{REQUEST_ID}` 占位符
3. 生成完整的日志平台查询链接

### 示例

**输入**：`request_id` = `b02df4aec7bd263a9b4f727eb605fad9`

**生成的链接**：
```
https://bklog.example.com/#/retrieve/481?projectId=6&bizId=7&start_time=now-7d&end_time=now&interval=auto&search_mode=sql&keyword=request_id%253A%2520%2522b02df4aec7bd263a9b4f727eb605fad9%2522
```

## 注意事项

1. **必须提供 request_id**：每次回答都必须明确告知用户该任务的 `request_id`，方便用户后续追溯
2. **链接格式正确性**：确保生成的链接中 `request_id` 被正确替换，URL 编码保持不变
3. **链接放在回答末尾**：日志平台查询链接应放在分析报告之后，作为补充信息提供给用户
4. **使用 Markdown 链接格式**：使用 `[链接文字](URL)` 格式，提升用户体验
5. **时间范围提示**：如果用户查询的任务时间较早（超过24小时），可以提示用户调整链接中的时间范围参数
